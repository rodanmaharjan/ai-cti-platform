# AI Usage Documentation

## Introduction

Generative AI tools were used throughout the development of this project to support brainstorming, coding assistance, debugging, documentation, and workflow improvement. The project followed an AI-assisted development approach where AI tools supported the implementation process while all final decisions, testing, and integration activities were performed manually.

---

# AI Tools Used

| AI Tool | Purpose |
|---|---|
| ChatGPT | Brainstorming, debugging assistance, code explanations, and documentation support |
| GitHub Copilot / AI Assistance | Code suggestions and syntax support |
| Streamlit Documentation | UI implementation guidance |
| VS Code AI Assistance | Development support and workflow improvement |

---

# AI-Assisted Workflow Activities

## 1. Project Brainstorming

Generative AI tools were used during the initial planning stage to:
- identify suitable project ideas,
- refine the project scope,
- and select a realistic cybersecurity use case.

Several project concepts were explored before selecting the IOC analysis dashboard because it was achievable within the available timeframe and aligned with practical cybersecurity workflows.

---

## 2. Code Development Assistance

AI tools were used to assist with:
- Python syntax,
- regular expression generation,
- Streamlit dashboard implementation,
- debugging,
- and UI improvement.

The development process remained incremental, where features were added and tested gradually.

Examples of AI-assisted implementation tasks included:
- IOC type detection logic,
- risk classification improvements,
- MITRE ATT&CK mapping integration,
- CSV export implementation,
- and code commenting for readability.

---

## 3. Debugging and Error Resolution

During development, Generative AI tools were used to:
- identify syntax errors,
- troubleshoot Streamlit issues,
- resolve package installation problems,
- and improve project structure.

This significantly reduced troubleshooting time and improved development efficiency.

---

## 4. Documentation Support

AI tools were also used to support:
- methodology writing,
- project log organisation,
- grammar improvement,
- formatting,
- and explanation of technical workflows.

The final documentation was manually reviewed and adjusted to ensure it accurately reflected the implemented project.

---

# Human Decision-Making and Validation

Although Generative AI tools assisted throughout development, all major implementation decisions were manually reviewed and validated. The project author selected the project scope, tested the functionality, verified outputs, and controlled the final implementation decisions.

The AI tools acted as development assistants rather than autonomous developers.

---

# Benefits of AI-Assisted Development

The use of AI-assisted workflows provided several advantages:
- faster development,
- easier debugging,
- improved code readability,
- simplified documentation creation,
- and improved productivity.

The workflow also demonstrated how AI tools can support practical cybersecurity development activities within realistic organisational environments.

---

# Conclusion

The project demonstrated that Generative AI tools can effectively support cybersecurity software development workflows when combined with manual validation, incremental prototyping, and human decision-making. The AI-assisted workflow improved efficiency while maintaining project control and implementation quality.
