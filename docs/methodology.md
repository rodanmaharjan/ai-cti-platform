# Methodology

## Introduction

This project was developed as part of the CI7526/30 Cyber and AI coursework module. The purpose of the project was to design and implement a simple AI-assisted cybersecurity tool capable of analysing Indicators of Compromise (IOCs) such as IP addresses, domains, and file hashes.

The project followed an incremental and AI-assisted development methodology. The system was gradually improved from a basic prototype into a more structured cybersecurity dashboard using Python, Streamlit, GitHub, and Generative AI tools.

---

## Project Planning and Initial Brainstorming

The initial planning stage involved identifying a realistic cybersecurity problem that could be implemented within the limited timeframe of the coursework. Existing Cyber Threat Intelligence (CTI) experience and a previously developed VirusTotal IOC checking script were used as the foundation for the project.

Several project ideas were considered, including:
- Threat intelligence automation
- IOC enrichment systems
- AI-assisted malware analysis
- Security reporting dashboards

A lightweight IOC analysis dashboard was selected because it was realistic, achievable within the coursework deadline, and aligned with practical CTI workflows.

---

## AI-Assisted Development Workflow

The project used an AI-assisted workflow throughout development. Generative AI tools were used to:
- brainstorm project ideas,
- improve project structure,
- assist with debugging,
- generate and improve code,
- explain Python functions,
- and refine documentation.

The development workflow involved:
1. Creating the initial project structure.
2. Developing a basic Streamlit dashboard.
3. Implementing IOC type detection using regular expressions.
4. Adding MITRE ATT&CK mappings.
5. Implementing risk classification logic.
6. Adding CSV report export functionality.
7. Improving the user interface and code readability.

The AI tools supported the development process by accelerating debugging, simplifying implementation tasks, and improving overall workflow efficiency.

---

## Development Approach

The project followed an incremental prototyping approach. Features were implemented gradually and tested after each improvement stage.

The first prototype contained:
- a basic Streamlit interface,
- IOC input functionality,
- and simple output display.

Later versions added:
- multi-IOC analysis,
- IOC classification,
- MITRE ATT&CK mapping,
- risk level identification,
- recommendations,
- and CSV export functionality.

This iterative process helped maintain a stable working prototype throughout development.

## Conclusion

The methodology used in this project allowed the system to be developed incrementally while maintaining a functional prototype throughout the coursework period. The combination of cybersecurity knowledge, Python development, and AI-assisted workflow support enabled the successful implementation of a simple IOC analysis dashboard suitable for cybersecurity analysis and reporting tasks.
